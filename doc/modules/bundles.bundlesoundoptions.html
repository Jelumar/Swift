Module bundles.bundlesoundoptions

Dieses Bundle bietet die Möglichkeit die Lautstärke im Spiel zu regeln.

Außerdem kannst du Stimmen abspielen und stoppen.

Anwenderfunktionen

API.PlaySound (_Sound, _PlayerID) Spielt einen Sound aus dem Spiel ab.
API.PlayVoice (_File) Gibt eine MP3-Datei als Stimme wieder.
API.SoundRestore () Stellt den Sound wieder her, sofern ein Backup erstellt wurde.
API.SoundSave () Erstellt ein Backup der Soundeinstellungen, wenn noch keins erstellt wurde.
API.SoundSetAtmoVolume (_Volume) Setzt die Lautstärke der Umgebung.
API.SoundSetMusicVolume (_Volume) Setzt die Lautstärke der Musik.
API.SoundSetUIVolume (_Volume) Setzt die Lautstärke des Interface.
API.SoundSetVoiceVolume (_Volume) Setzt die Lautstärke der Stimmen.
API.SoundSetVolume (_Volume) Setzt die allgemeine Lautstärke.
API.StartEventPlaylist (_Playlist, _PlayerID) Startet eine Playlist, welche als XML angegeben ist.
API.StopEventPlaylist (_Playlist, _PlayerID) Beendet eine Event Playlist.
API.StopVoice () Stoppt alle als Stimme abgespielten aktiven Sounds.

Anwenderfunktionen

API.PlaySound (_Sound, _PlayerID)
Spielt einen Sound aus dem Spiel ab.

Wenn eigene Sounds verwendet werden sollen, müssen sie im WAV-Format vorliegen und in die zwei Verzeichnisse für niedrige und hohe Qualität kopiert werden.

Verzeichnisstruktur für eigene Sounds: 

map_xyz.s6xmap.unpacked
|-- sounds/high/ui/*
|-- sounds/low/ui/*
|-- maps/externalmap/map_xyz/*
|-- ...

Parameters:

  • _Sound Pfad des Sound
  • _PlayerID (Optional) ID des menschlichen Spielers

Usage:

    API.PlaySound("ui\\menu_left_gold_pay");
API.PlayVoice (_File)
Gibt eine MP3-Datei als Stimme wieder. Diese Funktion kann auch benutzt werden um Geräusche abzuspielen.

Parameters:

  • _File string Abzuspielende Datei

Usage:

    API.PlayVoice("music/puhdys_alt_wie_ein_baum.mp3");

API.StartEventPlaylist (_Playlist, _PlayerID)
Startet eine Playlist, welche als XML angegeben ist.

Eine als XML definierte Playlist wird nicht als Voice abgespielt sondern als Music. Als Musik werden MP3-Dateien verwendet. Diese können entweder im Spiel vorhanden sein oder im Ordner music/ im Root-Verzeichnis des Spiels gespeichert werden. Die Playlist gehört ebenfalls ins Root- Verzeichnis nach config/sound/.

Verzeichnisstruktur für eigene Musik: 

map_xyz.s6xmap.unpacked
|-- music/*
|-- config/sound/*
|-- maps/externalmap/map_xyz/*
|-- ...

In der QSB sind bereits die Variablen gvMission.MusicRootPath und gvMission.PlaylistRootPath mit den entsprechenden Pfaden vordefiniert.

Wenn du eigene Musik verwendest, achte darauf, einen möglichst eindeutigen Namen zu verwenden. Und natürlich auch auf Urheberrecht!

Beispiel für eine Playlist: 

<?xml version="1.0" encoding="utf-8"?>
<PlayList>
 <PlayListEntry>
   <FileName>Music\some_music_file.mp3</FileName>
   <Type>Loop</Type>
 </PlayListEntry>
 <!-- Hier können weitere Einträge folgen. -->
</PlayList>
Als Typ können "Loop" oder "Normal" gewählt werden. Normale Musik wird einmalig abgespielt. Ein Loop läuft endlos weiter.

Außerdem kann zusätzlich zum Typ eine Abspielwahrscheinlichkeit mit angegeben werden: 

<Chance>10</Chance>
Es sind Zahlen von 1 bis 100 möglich.

Parameters:

  • _Playlist Pfad zur Playlist
  • _PlayerID (Optional) ID des menschlichen Spielers

Usage:

    API.StartEventPlaylist(gvMission.PlaylistRootPath .."my_playlist.xml");
