Module bundles.bundletravelingsalesman

Mit diesem Bundle wird ein Fahrender Händler angeboten der periodisch den Hafen mit einem Schiff anfährt.

Dabei kann der Fahrtweg frei mit Wegpunkten bestimmt werden. Es können auch mehrere Spieler zu Händlern gemacht werden.

API.TravelingSalesmanCreate (_Description)
Erstellt einen fahrender Händler mit zufälligen Angeboten.

Soll immer das selbe angeboten werden, darf es nur genauso viele Angebote geben, wie als Maximum gesetzt wird.

Es kann mehr als einen fahrender Händler auf der Map geben.

Angebote
Es können Waren, Soldaten oder Entertainer angeboten werden. Aus allen definierten Angeboten werden zufällig Angebote in der angegebenen Mange ausgesucht und gesetzt.

Routen
Um die Route anzugeben, wird ein Name eingegeben. Es werden alle fortlaufend nummerierte Punkte mit diesem Namen gesucht. Alternativ kann auch eine Liste von Punkten angegeben werden. Es muss mindestens 2 Punkte geben.

Alias: TravelingSalesmanActivate

Parameters:

  • _Description table Definition des Händlers

Usage:

    local TraderDescription = {
    PlayerID   = 2,       -- Partei des Hafen
    Path       = "SH2WP", -- Pfad (auch als Table einzelner Punkte möglich)
    Duration   = 150,     -- Ankerzeit in Sekunden (Standard: 360)
    Interval   = 3,       -- Monate zwischen zwei Anfarten (Standard: 2)
    OfferCount = 4,       -- Anzahl Angebote (1 bis 4) (Standard: 4)
    NoIce      = true,    -- Schiff kommt nicht im Winter (Standard: false)
    Offers = {
        -- Angebot, Menge
        {"G_Gems", 5},
        {"G_Iron", 5},
        {"G_Beer", 2},
        {"G_Stone", 5},
        {"G_Sheep", 1},
        {"G_Cheese", 2},
        {"G_Milk", 5},
        {"G_Grain", 5},
        {"G_Broom", 2},
        {"U_CatapultCart", 1},
        {"U_MilitarySword", 3},
        {"U_MilitaryBow", 3}
    },
};
API.TravelingSalesmanCreate(TraderDescription);
API.TravelingSalesmanDispose (_PlayerID)
Entfernt den fahrenden Händler von dem Spieler. Der Spieler bleibt erhalten wird aber nicht mal als fahrender Händler fungieren.

Hinweis: Wenn gerade ein Schiff unterwegs ist oder im Hafen liegt, wird es sofort gelöscht!

Alias: TravelingSalesmanDeactivate

Parameters:

  • _PlayerID number Spieler-ID des Händlers

Usage:

    API.TravelingSalesmanDispose(2);

Reward

Reward_TravelingSalesman (_PlayerID, _Interval, _Duration, _OfferCount1, _OfferType1, _OfferCount2, _OfferType2, _OfferCount3, _OfferType3, _OfferCount4, _OfferType4, _Path, _YieldIce)
Erstellt einen fliegenden Händler für den angegebenen Spieler.

Der fliegende Händler kann bis zu 4 Angebote haben. Schiffe werden im angegeben Interall anlegen und nach Ablauf der Aufenthaltsdauer wieder abfahren.

Parameters:

  • _PlayerID ID des Spielers
  • _Interval Interval in Monaten
  • _Duration Anlegezeit in Sekunden
  • _OfferCount1 Menge Angebot 1
  • _OfferType1 Typ Angebot 1
  • _OfferCount2 Menge Angebot 2
  • _OfferType2 Typ Angebot 2
  • _OfferCount3 Menge Angebot 3
  • _OfferType3 Typ Angebot 3
  • _OfferCount4 Menge Angebot 4
  • _OfferType4 Typ Angebot 4
  • _Path Präfix der Path Entities
  • _YieldIce Wenn Wasser gefriert pausieren
