Module addons.addonrandomrequests

Dieses Bundle stellt zufällige Aufgaben durch NPC-Spieler bereit.

Es gibt verschiedene Typen von zufälligen Aufträgen. Es wird versucht, immer einen einzigartigen Auftrag für den jeweiligen Auftraggeber zu erzeugen. Wenn kein Auftrag erzeugt werden kann, wird der Spieler aufgefordert 2000 Gold zu bezahlen.

Goal

Goal_RandomRequest (_Goods, _Gold, _Claim, _Title, _Reputation, _Walls, _Predator, _Mine, _Find, _Cure, _Object, _Build, _Time, _Repeat, _Suggestion, _Success, _Failure)
Wählt einen zufälligen Auftrag für den Spieler aus. Über die Parameter kann bestimmt werden, welche Typen von Aufträgen erscheinen können. Dieses Behavior sollte in versteckten Quests benutzt werden.

Tribute und Warenanforderungen steigen in der Menge mit höherem Titel des Auftragnehmers.

Dem Spieler können entweder keine Wiederholungen, eine feste Anzahl an Wiederholungen oder unendliche Wiederholungen gewährt werden. Das Behavior scheitert nur dann endgültig, wenn keine Versuche übrig sind.

Wichtig:

  • Die Option _Goods kann Rohstoffe wählen, die der Spieler wegen Ausgeblendetem Baumenü u.ä. nicht erlangen kann! In diesem Fall muss die Ware Auf die Blacklist des Auftraggebers gesetzt werden!
  • Die Option _Object benötigt interaktive Objekte, die "IORR" heißen und von 1 bis n fortlaufend nummeriert sind.
  • Die Option _Object erzeugt u.U. Raubtierspawnpoints (Tigerhöhle)!

Hinweis: Das Behavior erzeugt einen weiteren Quest mit dem zufällig gewählten Ziel. Somit ist es mit den Tribut-Quests vergleichbar. Wird Random Requests neu gestartet wird ein neuer Zufalls-Quest erstellt.

Hinweis: Es kann vorkommen, das bestimmte Auftragsarten unter gewissen Voraussetzungen fälschlich sofort als erfolgreich abgeschlossen gelten. (z.B. Monsun während Build Wall)

Parameters:

  • _Goods boolean Ziel: Waren liefern
  • _Gold boolean Ziel: Tribut bezahlen
  • _Claim boolean Ziel: Territorium erobern
  • _Title boolean Ziel: Nächst höherer Titel
  • _Reputation boolean Ziel: Ruf der Stadt
  • _Walls boolean Ziel: Stadt mit einer Mauer schützen
  • _Predator boolean Ziel: Raubtiere vernichten
  • _Mine boolean Ziel: Verschüttete Mine auffüllen
  • _Find boolean Ziel: Gegenstände suchen
  • _Cure boolean Ziel: Kranke Siedler heilen
  • _Object boolean Ziel: Zufällig gewähltes Objekt aktivieren
  • _Build boolean Ziel: Zufällig gewähltes Gebäude bauen
  • _Time number Zeit bis zur Niederlage (0 = aus)
  • _Repeat number Wiederholungen (-1 = endlos, 0 = aus)
  • _Suggestion string (optional) Startnachricht
  • _Success string (optional) Erfolgsnachricht
  • _Failure string (optional) Fehlschlagnachricht
