Module bundletravelingsalesman

Mit diesem Bundle wird ein Fahrender Händler angeboten der periodisch den Hafen mit einem Schiff anfährt.

Dabei kann der Fahrtweg frei mit Wegpunkten bestimmt werden. Es können auch mehrere Spieler zu Händlern gemacht werden.

API.TravelingSalesmanActivate (_PlayerID)
Aktiviert einen fahrenden Händler, der zuvor deaktiviert wurde.

Alias: TravelingSalesmanResume

Parameters:

  • _PlayerID number Spieler-ID des Händlers

Usage:

    API.TravelingSalesmanActivate(2);
API.TravelingSalesmanCreate (_Description)
Erstellt einen fahrender Händler mit zufälligen Angeboten.

Soll immer das selbe angeboten werden, darf nur ein Angebotsblock definiert werden. Es kann mehr als einen fahrender Händler auf der Map geben.

Angebote
Es sind maximal 4 Angebote pro Angebotsblock erlaubt. Es können Waren, Soldaten oder Entertainer angeboten werden. Es wird immer automatisch 1 Block selektiert und die Angebote gesetzt.

Routen
Über Waypoints wird der Weg bestimmt, den das Handelsschiff nehmen wird. Die Umkehrung der Route ist standardmäßig als Rückweg festgelegt. Will man einen anderen Rückweg, muss dieser als Reversed angegeben werden.

Alias: TravelingSalesmanActivate

Parameters:

  • _Description table Definition des Händlers

Usage:

    local TraderDescription = {
    PlayerID = 2,
    Waypoints = {"WP1", "WP2", "WP3", "WP4"},
    Offers = {
        {
            {"G_Gems", 5,},
            {"G_Iron", 5,},
            {"G_Beer", 2,},
        },
        {
            {"G_Stone", 5,},
            {"G_Sheep", 1,},
            {"G_Cheese", 2,},
            {"G_Milk", 5,},
        },
        {
            {"G_Grain", 5,},
            {"G_Broom", 2,},
            {"G_Sheep", 1,},
        },
        {
            {"U_CatapultCart", 1,},
            {"U_MilitarySword", 3,},
            {"U_MilitaryBow", 3,},
        },
    },
};

API.TravelingSalesmanDeactivate (_PlayerID)
Deaktiviert einen fahrenden Händler. Der aktuelle Zyklus wird noch beendet, aber danach kommt der Händler nicht mehr wieder.

Alias: TravelingSalesmanYield

Parameters:

  • _PlayerID number Spieler-ID des Händlers

Usage:

    API.TravelingSalesmanDeactivate(2);
API.TravelingSalesmanDiplomacyOverride (_PlayerID, _Flag)
Legt fest, ob die diplomatischen Beziehungen zwischen dem Spieler und dem Hafen überschrieben werden.

Die diplomatischen Beziehungen werden überschrieben, wenn sich ein Schiff im Hafen befindet und wenn es abreist. Der Hafen ist "Handelspartner", wenn ein Schiff angelegt hat, sonst "Bekannt".

Bei diplomatischen Beziehungen geringer als "Bekannt", kann es zu Fehlern kommen. Dann werden Handelsangebote angezeigt, konnen aber nicht durch den Spieler erworben werden.

Hinweis: Überschreiben der Beziehungen ist Standardmäßig als aktiv voreingestellt.

Alias: TravelingSalesmanDiplomacyOverride

Parameters:

  • _PlayerID number Spieler-ID des Händlers
  • _Flag boolean Diplomatie überschreiben

Usage:

    API.TravelingSalesmanDiplomacyOverride(2, false);
API.TravelingSalesmanDispose (_PlayerID)
Entfernt den fahrenden Händler von dem Spieler. Der Spieler bleibt erhalten wird aber nicht mal als fahrender Händler fungieren.

Alias: TravelingSalesmanDeactivate

Parameters:

  • _PlayerID number Spieler-ID des Händlers

Usage:

    API.TravelingSalesmanDispose(2);
